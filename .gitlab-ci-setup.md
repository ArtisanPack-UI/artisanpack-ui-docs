# GitLab CI/CD Setup Guide

This guide explains how to set up the GitLab CI/CD pipeline for this Laravel application with Laravel Cloud deployment.

## Pipeline Overview

The pipeline has 4 stages:

1. **Build** - Install dependencies (Composer & NPM)
2. **Test** - Run unit and feature tests with Pest
3. **Quality** - Run code style checks (Pint) and security audits
4. **Deploy** - Deploy to Laravel Cloud (production/staging)

## Prerequisites

- GitLab account with repository access
- Laravel Cloud project set up
- GitLab Runner configured (or use shared runners)

## Required GitLab CI/CD Variables

Configure these variables in your GitLab project:
**Settings → CI/CD → Variables**

### Optional (for API-based deployment control):

| Variable | Value | Protected | Masked |
|----------|-------|-----------|--------|
| `LARAVEL_CLOUD_API_TOKEN` | Your Laravel Cloud API token | ✓ | ✓ |
| `LARAVEL_CLOUD_PROJECT_ID` | Your production project ID | ✓ | - |
| `LARAVEL_CLOUD_STAGING_PROJECT_ID` | Your staging project ID | ✓ | - |

### How to get these values:

1. **Laravel Cloud API Token:**
   - Log in to Laravel Cloud
   - Go to Settings → API Tokens
   - Create a new token with deployment permissions
   - Copy the token (you won't see it again)

2. **Project IDs:**
   - Go to your Laravel Cloud dashboard
   - Click on your project
   - The project ID is in the URL: `cloud.laravel.com/projects/{PROJECT_ID}`

## Laravel Cloud Integration

### Option 1: Auto-Deploy with Pipeline Gating (Recommended)

Laravel Cloud can auto-deploy on push, but only **after** GitLab CI pipeline passes.

**In Laravel Cloud:**
1. Go to your project settings
2. Under "Deployments" → "Auto Deploy"
3. Set the branch to `main`
4. Enable "Wait for CI/CD" option
5. Select "GitLab" as the CI provider

This configuration ensures:
- Push to `main` → GitLab CI runs tests
- If tests pass → Laravel Cloud auto-deploys
- If tests fail → Deployment is blocked

### Option 2: Manual API-Based Deployment

If you prefer full control, uncomment the API deployment code in `.gitlab-ci.yml`:

```yaml
deploy:production:
  script:
    # Uncomment these lines:
    - |
      curl -X POST "https://cloud.laravel.com/api/projects/${LARAVEL_CLOUD_PROJECT_ID}/deployments" \
        -H "Authorization: Bearer ${LARAVEL_CLOUD_API_TOKEN}" \
        -H "Content-Type: application/json" \
        -d "{\"commit\": \"${CI_COMMIT_SHA}\"}"
```

This gives you complete control over when deployments happen via GitLab CI.

## Branch Strategy

The pipeline is configured for:

- **`main` branch:**
  - Runs all tests and quality checks
  - Deploys to **production** (manual approval required)

- **`develop` branch:**
  - Runs all tests and quality checks
  - Auto-deploys to **staging** (if tests pass)

- **Merge Requests:**
  - Runs all tests and quality checks
  - No deployment

## Pipeline Behavior

### On Push to `main`:
1. Build stage installs dependencies
2. Test stage runs all tests in parallel
3. Quality stage checks code style and security
4. Deploy stage waits for manual approval
5. After approval, deployment proceeds (or Laravel Cloud auto-deploys)

### On Push to `develop`:
1-3. Same as above
4. Deploys automatically to staging if all checks pass

### On Merge Request:
1-3. Same as above
4. No deployment (just validation)

## Debugging Failed Pipelines

### Tests Failing?
```bash
# Run tests locally first
php artisan test

# Check specific test
php artisan test --filter=test_name

# Check for missing migrations
php artisan migrate:status
```

### Pint Failing?
```bash
# Run Pint locally
./vendor/bin/pint --test

# Auto-fix issues
./vendor/bin/pint
```

### Build Failing?
- Check composer.json is valid: `composer validate`
- Check package.json is valid: `npm ci`
- Verify all dependencies are committed

## Disabling Laravel Cloud Auto-Deploy

If you want GitLab CI to be the **only** way to deploy:

1. In Laravel Cloud project settings
2. Go to "Deployments" → "Auto Deploy"
3. **Disable** auto-deploy
4. Uncomment the API deployment code in `.gitlab-ci.yml`
5. Set up the required environment variables

This ensures deployments **only** happen through GitLab CI pipeline.

## Security Notes

- Never commit `.env` file
- All API tokens should be stored in GitLab CI/CD variables (masked)
- Use "Protected" variables for production credentials
- Review the security audit report before deploying

## Monitoring Deployments

### GitLab:
- View pipeline status: **CI/CD → Pipelines**
- View deployment history: **Deployments → Environments**

### Laravel Cloud:
- View deployment logs in your project dashboard
- Set up notifications for failed deployments
- Monitor application health and errors

## Troubleshooting

### Pipeline is slow?
- GitLab caches dependencies between runs
- Use GitLab Runner with more resources
- Consider using Docker layer caching

### Laravel Cloud not deploying after pipeline passes?
1. Check Laravel Cloud webhook settings
2. Verify "Wait for CI/CD" is enabled
3. Check Laravel Cloud deployment logs for errors

### Manual deployment needed immediately?
You can trigger a deployment manually in Laravel Cloud dashboard, bypassing the pipeline.
